# Binoculars (Hans et al., 2024) — Reproduction Notes

This folder is for running the Binoculars zero-shot detector: it scores a passage by comparing how surprising it is to an “observer” language model versus how much two closely related models disagree, using an observer-perplexity / cross-perplexity ratio.

## Environment setup

Binoculars was run in Google Colab. No local environment setup is required unless you want to reproduce the run outside Colab.

If you are reproducing in Colab, the typical setup is:

1. Open a Colab notebook and set the runtime to use a GPU.
3. Load the evaluation inputs (e.g., from `HC3-dataset-samples/`) and upload evaluate_samples.py to produce per-sample scores and aggregated metrics.

## Device / system used

This is the environment used for our Binoculars evaluation:

- Platform: Google Colab
- SuggestedGPU: NVIDIA A100 (High-RAM)
- Runtime: ~1.5 hours for the evaluation run

## How to run

Our Binoculars evaluation was run from a Colab upload that included:

- a dataset CSV (e.g., `hc3_10000.csv` generated from `HC3-dataset-samples/`), and
- an evaluation script, `evaluate_samples.py` and load the CSV to compute Binoculars scores/metrics.

Typical workflow:

1. Prepare inputs: a set of human and model-generated passages (see the HC3 samples folder).
2. Compute a Binoculars score per passage using the chosen observer/performer model pair.
3. Aggregate scores into low-FPR metrics (e.g., TPR at a fixed FPR) and/or a ROC curve.

## How results are generated

Results should be generated by:

1. Selecting an observer/perfomer pair (in our configuration: Falcon-7B as the observer and Falcon-7B-Instruct as the performer, matching the Binoculars paper defaults).
2. For each passage, computing:
   - observer perplexity (how surprising the passage is under the observer model), and
   - cross-perplexity (how well the observer model matches the performer model’s next-token distributions across positions),
   then forming the ratio-style score.
3. Sweeping a threshold over scores to compute detection metrics, including operating points in the low-FPR regime (e.g., `0.1%` FPR).
